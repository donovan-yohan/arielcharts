## 1. Project Overview

ArielCharts is a real-time collaborative Mermaid diagram editor. Humans and AI agents edit the same diagram simultaneously — humans via a browser-based editor, agents via MCP tools.

### Architecture

pnpm monorepo with three packages:

- **`apps/web`** — Next.js frontend (TypeScript, CodeMirror 6, Yjs CRDT, mermaid v11, React)
- **`apps/server`** — Node.js + Express backend (TypeScript, WebSocket via y-websocket, MCP SDK, SQLite persistence)
- **`packages/shared`** — Shared types and constants

### Core Principles

1. **Mermaid text is the single source of truth.** All state derives from the Yjs `Y.Text` document.
2. **Flowchart first.** Other diagram types degrade to read-only SVG preview.
3. **Two-way sync.** Visual edits serialize back to text; text edits update the visual overlay.
4. **Roundtrip fidelity over formatting.** Text must not get mangled by visual edits.

### Dependencies

- `mermaid` v11.12.0 — diagram rendering
- `mermaid-ast` v0.8.2 — typed AST parsing and round-trip serialization for flowcharts
- `fast-diff` — minimal character-level diffs for Yjs operations
- `lucide-react` — icons
- `yjs`, `y-websocket`, `y-codemirror.next` — CRDT collaboration
- `@modelcontextprotocol/sdk` — MCP server (Streamable HTTP transport)

## 3. Interactive Diagram Editor

### Approach: Mermaid SVG + Structured React Overlay

Mermaid renders the SVG. After render, walk the SVG DOM to build a hit map of node bounding boxes. React overlay components (toolbars, editors, connection handles) are positioned absolutely over the SVG. Mermaid owns rendering; React owns interaction.

### Architecture

```
Yjs Text (CRDT source of truth)
  |
  +-- CodeMirror Editor (left pane, yCollab)
  |
  +-- MutationQueue (visual edits)
  |     +-- mermaid-ast: parse() -> mutate AST -> render()
  |     +-- fast-diff: old text vs new text -> minimal char ops
  |     +-- Yjs: insert/delete only changed characters
  |
  +-- mermaid.render() -> SVG
        |
        +-- SVG Hit Map (getCTM-based coordinate transform)
        |
        +-- React Overlay (inside same CSS transform wrapper)
              +-- Transparent click targets
              +-- Selection outlines (blue glow)
              +-- Contextual toolbar (floating above node)
              +-- Inline text editor
              +-- Shape picker dropdown
              +-- Connection ports + rubber-band line
              +-- Mode hint pill
              +-- Zoom controls cluster
```

### SVG Hit Map (`lib/svg-hit-map.ts`)

After mermaid renders SVG, query the DOM to build a map of interactive regions:

```typescript
interface SvgHitMap {
  nodes: Map<string, SvgBounds>;      // nodeId -> bounding box in CSS pixels
  edges: Map<string, SvgEdgeHit>;     // edgeKey -> bounds + SVG path element
  subgraphs: Map<string, SvgBounds>;  // subgraphId -> bounding box
  viewBox: SvgViewBox;
}
```

Mermaid v11 SVG structure:
- Nodes: `<g class="node" id="flowchart-{nodeId}-{index}">` with `transform="translate(x,y)"`
- Edges: `<g class="edgePath">` containing `<path>` elements
- Subgraphs: `<g class="cluster">` with nested content

**Critical implementation detail:** Uses `element.getCTM()` (Current Transform Matrix) to transform local `getBBox()` coordinates through the full SVG transform chain to CSS pixel coordinates. Raw `getBBox()` returns center-origin local coordinates (e.g., `{x: -48, y: -27}`) that don't account for parent `<g transform="translate(...)">` wrappers. The hit map builds from the actually-rendered SVG element inside `DiagramCanvas` via `requestAnimationFrame`.

### Infinite Canvas (`components/diagram-canvas.tsx`)

Figma-style pan/zoom canvas wrapping the SVG + overlay:

- **Scroll wheel:** zoom in/out centered on cursor
- **Space+drag or middle-click drag:** pan
- **Double-click empty space:** fit diagram to viewport
- **Zoom range:** 10% to 400%
- Implementation: CSS `transform: translate(panX, panY) scale(zoom)` with `transform-origin: 0 0`
- Both SVG and overlays share the same transform wrapper

**Zoom-to-cursor math:**
```
canvasX = (clientX - panX) / zoom
canvasY = (clientY - panY) / zoom
newZoom = clamp(zoom * (deltaY > 0 ? 0.9 : 1.1), 0.1, 4.0)
panX = clientX - canvasX * newZoom
panY = clientY - canvasY * newZoom
```

### Interaction Modes

**Select mode (default):**
- Click node: select, show contextual toolbar
- Double-click node: open inline text editor
- Shift+click: multi-select
- Click empty space: deselect all
- Delete/Backspace: delete selected node(s)
- Cmd+G: group selected nodes into subgraph (prompts for label)
- Cmd+Shift+G: ungroup selected subgraph

**Connect mode (from toolbar):**
- Click source node → click target node → edge created
- Rubber-band line follows cursor from source
- Edge label prompt on midpoint after connecting
- Arrow type selector in toolbar
- Escape cancels

### Mutation Layer (`lib/diagram-mutations.ts`)

All visual edits flow through a `MutationQueue` that serializes operations and applies minimal diffs to Yjs:

```typescript
class MutationQueue {
  enqueue(mutate: (currentText: string) => string) {
    // 1. Read latest Yjs text
    // 2. Parse -> mutate AST -> serialize via mermaid-ast
    // 3. fast-diff -> minimal Yjs insert/delete ops
  }
}
```

Mutation wrappers: `editNodeLabel`, `changeNodeShape`, `addNode`, `removeNode`, `addEdge`, `removeEdge`, `groupNodes`, `ungroupSubgraph`

**Note:** Users edit display labels only, never node IDs. The first visual edit normalizes formatting (mermaid-ast canonical form); subsequent edits produce minimal diffs.

### Two-Way Sync

- **Visual edit:** MutationQueue parses current text → mutates AST → serializes → fast-diff → Yjs ops → mermaid re-renders → hit map rebuilds
- **Text edit:** Yjs observer fires → mermaid re-renders → hit map rebuilds → overlay repositions
- **Debounce:** re-parse/re-render at ~150ms to avoid thrashing during rapid typing

### Visual Design

**Contextual Toolbar:** Dark pill floating above selected node. Icons: pencil (edit), shapes (change shape), arrow (connect), trash (delete), plus (add node). 16px Lucide icons, 24x24px touch targets.

**Shape Picker:** 2-column grid below toolbar, 10 shapes. Active shape highlighted with `#38bdf8` border.

**Inline Text Editor:** Double-click node opens input field. `#38bdf8` underline, auto-select, Enter/blur commits, Escape cancels.

**Connection Mode:** Crosshair cursor, 6px `#38bdf8` port circles on nodes, dashed rubber-band line, edge label prompt on midpoint.

### Keyboard Accessibility

- **Tab** into canvas focuses first node
- **Arrow keys** move focus between connected nodes (follows edges)
- **Enter** on focused node opens toolbar
- **Escape** closes toolbar → deselects → returns focus to canvas
- Canvas: `role="application"`. Nodes: `role="button"`, `aria-label="[shape]: [label]"`

### Edge Cases

- **Invalid mermaid text:** Overlay disappears, last valid SVG shown + error banner
- **Non-flowchart:** Read-only SVG, no overlay
- **Empty diagram:** Centered "Add your first node" button
- **Concurrent edits:** Text editor buffers locally, commits on blur/Enter using latest graph. If edited node deleted remotely, discard and close.
- **Zoom threshold:** Close editors below 40% zoom

## 5. Shared Contracts

Source of truth: `packages/shared/src/types.ts`

### Core Types

```typescript
interface Participant {
  name: string;
  color: string;
  type: 'human' | 'agent';
}

interface AwarenessState {
  user: Participant;
  cursor?: { anchor: number; head: number };
}

interface ActivityEvent {
  id: string;
  timestamp: number;
  actor: { name: string; type: 'human' | 'agent' };
  action: 'joined' | 'left' | 'edited' | 'replaced';
  detail?: string;
}
```

### MCP Tool Shapes

```typescript
interface ReadDiagramInput  { session_id: string }
interface ReadDiagramOutput { mermaid_text: string; participants: Participant[] }
interface WriteDiagramInput { session_id: string; mermaid_text: string }
interface WriteDiagramOutput { success: boolean }
interface ListSessionsOutput { sessions: { id: string; title: string; participants: number }[] }
```

### WebSocket Contract

- Endpoint: `/ws/:roomId`
- Protocol: y-websocket-compatible framing (sync `messageType=0`, awareness `messageType=1`, query-awareness `messageType=3`)
- Initial: server sends sync step 1 + current awareness snapshot

### MCP HTTP Contract

- Uses `@modelcontextprotocol/sdk` with `StreamableHTTPServerTransport` on `/mcp`
- Origin validation via `ALLOWED_ORIGINS` env var
- CORS headers emitted; `OPTIONS /mcp` returns 204 for preflight

Agent config:
```json
{
  "mcpServers": {
    "arielcharts": {
      "type": "streamable-http",
      "url": "https://arielcharts-server.fly.dev/mcp"
    }
  }
}
```

### Coordination Rules

- Backend owns: Yjs doc lifecycle, WebSocket transport, persistence, session cleanup, MCP tools
- Frontend owns: shared types consumption, session routing, awareness/activity rendering
- `write_diagram` must append an ActivityEvent and update persisted Yjs state
- `list_sessions` returns live + persisted sessions ordered by `updatedAt`
- Session ID format: lowercase letters, digits, `_`, `-`, length 6-32

## 6. Known Risks & Deferred Work

### Known Risks

1. **SVG stability across mermaid versions.** Mermaid's SVG output structure (class names, ID patterns) isn't a public API. Pin mermaid version; integration tests verify SVG structure.
2. **Session persistence.** Currently rewrites full encoded Yjs doc on each update. Acceptable for MVP.
3. **Participant uniqueness.** Name-keyed, so duplicate display names can collapse.

### NOT in Scope

- Mobile/tablet responsive layout (desktop-only tool)
- Editing existing edge labels from the visual editor (use text editor)
- Drag-and-drop node positioning (mermaid layout is deterministic)
- Undo/redo UI (already works via Y.UndoManager + CodeMirror keybindings)
- Non-flowchart interactive editing (sequence, class, state diagrams are read-only)
